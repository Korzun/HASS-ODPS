# Design: Kebab-case File Naming & Co-located Tests

**Date:** 2026-04-18
**Status:** Approved

## Summary

Two related housekeeping changes:
1. Rename all PascalCase source files to kebab-case.
2. Move all tests from the separate `tests/` directory into `app/` alongside the source files they test.

## Scope

### Source file renames

Only 3 files currently use PascalCase; all others are already lowercase and need no change.

| Before | After |
|--------|-------|
| `app/services/BookStore.ts` | `app/services/book-store.ts` |
| `app/services/EpubParser.ts` | `app/services/epub-parser.ts` |
| `app/services/UserStore.ts` | `app/services/user-store.ts` |

### Test file moves

All 8 test files move from the flat `tests/` directory into `app/` alongside their source:

| Before | After |
|--------|-------|
| `tests/BookStore.test.ts` | `app/services/book-store.test.ts` |
| `tests/EpubParser.test.ts` | `app/services/epub-parser.test.ts` |
| `tests/UserStore.test.ts` | `app/services/user-store.test.ts` |
| `tests/kosync.test.ts` | `app/routes/kosync.test.ts` |
| `tests/opds.test.ts` | `app/routes/opds.test.ts` |
| `tests/ui.test.ts` | `app/routes/ui.test.ts` |
| `tests/users.test.ts` | `app/routes/users.test.ts` |
| `tests/logger.test.ts` | `app/logger.test.ts` |

The `tests/` directory is deleted after all files are moved.

## Execution approach

Use `git mv` for every rename and move so git history and blame are preserved. Then apply targeted edits to fix import paths.

## Changes required beyond file moves

### `jest.config.js`

Update `roots` to point at `app/` instead of `tests/`:

```js
// before
roots: ['<rootDir>/tests']
// after
roots: ['<rootDir>/app']
```

### Import paths in source files

Any source file importing from the 3 renamed services must update the path suffix:

- `'../services/BookStore'` → `'../services/book-store'`
- `'../services/EpubParser'` → `'../services/epub-parser'`
- `'../services/UserStore'` → `'../services/user-store'`

### Import paths in test files

Test files that import their subject using the old path (e.g., `'../../app/services/BookStore'`) must be updated to relative imports matching the new co-located location (e.g., `'./book-store'`).

## Out of scope

- `tsconfig.json` — uses directory globs (`app/**/*`), unaffected.
- `package.json` build script — references `app/public`, unaffected.
- `dist/` — auto-generated by `tsc`, no manual changes needed.
- Any refactoring of test content or source logic.

# CI: Testing & Linting on PRs and Main

**Date:** 2026-04-19  
**Status:** Approved

## Summary

Add a GitHub Actions CI workflow that runs ESLint (with TypeScript and Prettier) and Jest tests in parallel on every PR and push to `main`. Both checks are required status checks — PRs cannot be merged unless both pass.

## Workflow

**File:** `.github/workflows/ci.yml`

**Triggers:**
- `push` to `main`
- `pull_request` targeting `main`

**Jobs (parallel, `ubuntu-latest`, Node 24):**

| Job | Steps |
|---|---|
| `lint` | checkout → setup Node 24 → `npm ci` → `npm run lint` |
| `test` | checkout → setup Node 24 → `npm ci` → `npm run test` |

No `needs` dependency between jobs — they run concurrently.

**Branch protection** (manual, post-deploy): require both `lint` and `test` status checks in GitHub repository settings.

## ESLint Setup

**New devDependencies:**
- `eslint` — core linter
- `typescript-eslint` — TypeScript parser and rules (unified modern package)
- `eslint-config-prettier` — disables ESLint formatting rules that conflict with Prettier
- `prettier` — code formatter

**Config file:** `eslint.config.js` (ESLint flat config format)

**New `package.json` scripts:**
- `"lint"`: `eslint . && tsc --noEmit` — linting + TypeScript type check
- `"lint:fix"`: `eslint . --fix` — auto-fix lint issues

## Out of Scope

- Prettier as a standalone script/pre-commit hook (enforced through ESLint only)
- Matrix builds across multiple Node versions
- Caching `node_modules` between runs (can be added later if build times become a concern)

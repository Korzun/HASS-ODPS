# EPUBCheck Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Validate EPUBs with `epubcheck-ts` on upload (reject invalid before storing) and on metadata edit (verify the rewritten file before it replaces the stored book), blocking only on `ERROR`/`FATAL`.

**Architecture:** Add `epubcheck-ts` as a git submodule wired into the npm workspace. A thin `epub-validator` adapter wraps `validateEpub` and throws `EpubValidationError` when a report is not `valid`. Two route enforcement points consume it; the metadata-edit path is refactored to build the new EPUB into a `Buffer`, validate it, and only then atomically replace the stored file.

**Tech Stack:** TypeScript (CommonJS server), Express, multer, adm-zip, `epubcheck-ts` (ESM dual-format), Jest + ts-jest + supertest, Docker multi-stage, GitHub Actions.

## Global Constraints

- All work on the `add-epubcheck` branch — never commit to `main`.
- Blocking policy is **errors only**: reject when `report.valid === false` (i.e. any `FATAL` or `ERROR`). `WARNING`/`INFO`/`USAGE` never block.
- `epubcheck-ts` API: `validateEpub(input: Uint8Array | ArrayBuffer | ReadableStream<Uint8Array>, options?): Promise<Report>` where `Report = { messages: Message[]; epubVersion?: '2.0'|'3.0'; counts: Record<Severity, number>; fatal: boolean; valid: boolean }` and `Severity = 'FATAL'|'ERROR'|'WARNING'|'INFO'|'USAGE'`.
- React/component and service files use **kebab-case**.
- Lint runs from repo root: `npm run lint`. Per-workspace tests: `npm test -w app/server -- <pattern>`.
- Status codes: upload-invalid → **400**; edit-invalid → **422**.

## Prerequisite Gate (Task 1)

Pin the submodule to an `epubcheck-ts` commit that ships the **dual-format (ESM + CJS)** build (the `exports` map has a `require` condition and `dist/index.cjs` + `dist/index.d.ts` exist). The CommonJS server and ts-jest cannot consume an ESM-only build. If that commit does not yet exist upstream, stop and land the dual-format change in `epubcheck-ts` first; every later task depends on its built `dist/` (types at compile time, CJS entry at runtime).

- [ ] **Step 3: Write the adapter**

Create `app/server/services/epub-validator.ts`:

```ts
import { validateEpub } from 'epubcheck-ts';
import type { Report, Message, Severity } from 'epubcheck-ts';

export interface ValidationMessage {
  id: string;
  severity: Severity;
  message: string;
  location?: string;
}

const BLOCKING: ReadonlySet<Severity> = new Set<Severity>(['FATAL', 'ERROR']);

export function formatMessages(messages: Message[]): ValidationMessage[] {
  return messages.map((m) => ({
    id: m.id,
    severity: m.severity,
    message: m.message,
    location: m.location ? String(m.location.path ?? '') || undefined : undefined,
  }));
}

export class EpubValidationError extends Error {
  readonly messages: ValidationMessage[];
  readonly counts: Record<Severity, number>;

  constructor(messages: ValidationMessage[], counts: Record<Severity, number>) {
    super(`EPUB failed validation: ${counts.FATAL} fatal, ${counts.ERROR} error(s)`);
    this.name = 'EpubValidationError';
    this.messages = messages;
    this.counts = counts;
  }
}

export async function assertValidEpub(bytes: Buffer): Promise<Report> {
  const report = await validateEpub(bytes);
  if (!report.valid) {
    const blocking = formatMessages(report.messages.filter((m) => BLOCKING.has(m.severity)));
    throw new EpubValidationError(blocking, report.counts);
  }
  return report;
}
```

> Note: `Message.location`'s exact shape comes from `epubcheck-ts`. If `tsc` reports a type error on `m.location.path`, adjust `formatMessages` to the real `Location` fields (check `vendor/epubcheck-ts/dist/index.d.ts`). The blocking/throw logic does not depend on it.
